# Portfolio - Vision et Pilotage

Ce README decrit le portfolio au niveau produit: vision, priorites, contenus attendus et ordre des projets.

Le README technique de l'application web se trouve dans:
[portfolio-website/README.md](portfolio-website/README.md)

## Role de ce dossier

- Definir le cadre global du portfolio
- Garder une roadmap stable des projets a publier
- Centraliser les objectifs metier et les livrables

## Objectif global

Construire un portfolio qui permet de comprendre rapidement:
- le profil,
- les competences,
- les projets publies,
- la diversite technique du travail realise.

## Livrable principal

- Une vitrine web publiee, simple et lisible

## Positionnement du site portfolio

- Le site principal est une vitrine, pas un produit metier autonome
- Il n'a pas besoin d'un backend dedie a ce stade
- Il doit rester facile a maintenir, rapide a lire et rapide a deployer
- Il pourra evoluer plus tard si un besoin concret apparait

## Roadmap projets

1. Portfolio Website MVP
2. Application web full stack PWA
3. Application React Router + Spring Boot + MongoDB Atlas
4. Jeu ou prototype interactif Godot

## Projets a integrer dans la vitrine

### 1) Application web full stack PWA
- Stack cible: Next.js, TypeScript, Prisma, PostgreSQL, Vercel
- Objectif: demonstrer une architecture web moderne de bout en bout dans une application unifiee

### 2) Application a architecture separee
- Stack cible: React Router, Spring Boot, MongoDB Atlas
- Objectif: demonstrer une separation claire entre frontend SPA et backend API Java

### 3) Jeu ou prototype interactif
- Stack cible: Godot, C#, Blender
- Objectif: demonstrer interactivite, logique logicielle et production d'assets

## Planning initial

### Semaine 0
- Finaliser la vitrine portfolio
- Verrouiller les 3 projets et leur scope MVP

### Suite
- Publier chaque projet des qu'un MVP montrable existe
- Integrer ensuite liens demo, repo et visuels dans la vitrine

## Gouvernance de mise a jour
