# Portfolio - Vision et Pilotage

Ce README décrit le portfolio au niveau produit: vision, priorités, contenus attendus et ordre des projets.

Le README technique de l'application web se trouve dans:
[portfolio-website/README.md](portfolio-website/README.md)

## Rôle de ce dossier

- Définir le cadre global du portfolio
- Garder une roadmap stable des projets à publier
- Centraliser les objectifs métier et les livrables

## Objectif global

Construire un portfolio qui permet de comprendre rapidement:
- le profil,
- les compétences,
- les projets publiés,
- la diversité technique du travail réalisé.

## Livrable principal

- Une vitrine web publiée, simple et lisible

## Positionnement du site portfolio

- Le site principal est une vitrine, pas un produit metier autonome
- Il n'a pas besoin d'un backend dédié à ce stade
- Il doit rester facile à maintenir, rapide à lire et rapide à déployer
- Il pourra évoluer plus tard si un besoin concret apparaît
- La page d'accueil doit d'abord montrer des projets déjà réalisés avec preuves publiques avant la roadmap future

## Roadmap projets

1. Portfolio Website MVP
2. Application web full stack PWA
3. Application React Router + Spring Boot + MongoDB Atlas
4. Jeu ou prototype interactif Godot

## Projets à intégrer dans la vitrine

### 1) Application web full stack PWA
- Stack cible: Next.js, TypeScript, Prisma, PostgreSQL, Vercel
- Objectif: démontrer une architecture web moderne de bout en bout dans une application unifiée

### 2) Application à architecture séparée
- Stack cible: React Router, Spring Boot, MongoDB Atlas
- Objectif: démontrer une séparation claire entre frontend SPA et backend API Java

### 3) Jeu ou prototype interactif
- Stack cible: Godot, C#, Blender
- Objectif: démontrer interactivité, logique logicielle et production d'assets

## Planning initial

### Semaine 0
- Finaliser la vitrine portfolio
- Verrouiller les 3 projets et leur scope MVP

### Suite
- Publier chaque projet dès qu'un MVP montrable existe
- Intégrer ensuite liens démo, repo et visuels dans la vitrine
- Mettre en avant les projets déjà démontrables sur la page d'accueil avant les projets encore en cours

## Gouvernance de mise a jour

- Ajouter chaque projet dès qu'il devient publiable
- Remplacer le badge En cours par Live dès qu'une démo est disponible
- Mettre à jour les liens GitHub et démo en continu
- Garder une cohérence stricte entre Overview, to-do, MVP et contenu du site
